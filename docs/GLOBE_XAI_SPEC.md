### 3.2 시간 범위

- **전체**: 기원전 2333년(고조선 건국) ~ 2025년(현대)
- **시대 구분**: 고대(~476) → 중세(476~1453) → 근세(1453~1789) → 근대(1789~1945) → 현대(1945~)

### 3.3 공간 범위

| 지역 코드 | 명칭 | 데이터 건수 |
|-----------|------|-----------|
| korea | 한국 | 92 |
| europe | 유럽 | 130 |
| eastAsia | 동아시아 | 33 |
| americas | 아메리카 | 48 |
| middleEast | 중동 | 6 |
| africa | 아프리카 | 6 |
| southAsia | 남아시아 | 11 |

### 3.4 좌표·경계·연표 기준

- **좌표계**: WGS84 (위도·경도) — CesiumJS 표준
- **좌표 출처**: 각 사건의 대표적 발생 지점 또는 인물의 출생지를 기준으로 수동 지정
- **날짜 형식**: ISO 8601 확장 (`-2333-01-01` ~ `2025-01-01`)
  - 음수 연도는 기원전(BC)을 의미

### 3.5 자료 불확실성 처리

- 정확한 좌표가 불분명한 경우 → 해당 지역의 **대표 좌표**(수도, 중심지)를 사용
- 연도만 확인되고 월일이 불확실한 경우 → `01-01`로 기본 설정
- **학습자 설명**: "정확한 위치나 날짜가 확인되지 않는 사건은 해당 지역의 대표 지점과 연도의 시작일로 표시됩니다."

### 3.6 갱신·검증 원칙

1. 새 데이터 추가 시 `id`, `coordinates`, `era`, `region` 필드의 정합성을 검증한다.
2. 좌표 변경 시 지구본 위에서 시각적으로 위치를 확인한다.
3. 시대 구분(`era`) 변경 시 `categories.js`의 `ERAS.range`와 일치 여부를 확인한다.

## 4. 시각화 요소 정의 (Single Source of Truth)

### 4.1 마커 (Point Marker)

#### ✅ 역사·지리 개념 정의

- **주제**: 역사 사건 또는 인물의 공간적 위치를 지구본 위에 점(point)으로 표현
- **공간 범위**: 사건의 `location.coordinates` 또는 인물의 `birthPlace.coordinates`

#### ✅ 시간·공간 표현 로직

| 항목 | 구현 기준 |
|------|----------|
| 좌표 변환 | `Cesium.Cartesian3.fromDegrees(lng, lat)` — WGS84 경위도를 3D 직교좌표로 변환 |
| 높이 참조 | `HeightReference.CLAMP_TO_GROUND` — 지표면에 밀착 |
| 깊이 테스트 | `disableDepthTestDistance: POSITIVE_INFINITY` — 지구 뒤편에서도 항상 표시 |
| 마커 크기 | 10px 고정 (줌 레벨 무관) |
| 마커 색상 | 시대(`era`)에 따른 `getEraColor()` 반환값 적용 |

**시대별 색상 체계** (개발자 기준):

| 시대 | 색상 코드 | 시각적 의미 |
|------|----------|-----------|
| 고대 (ancient) | `#C9A96E` (황금) | 고대 문명의 토기·금속 이미지 |
| 중세 (medieval) | `#8B5E3C` (갈색) | 중세 양피지·성곽의 이미지 |
| 근세 (earlyModern) | `#4A7C59` (녹색) | 르네상스·대항해의 자연 탐험 이미지 |
| 근대 (modern) | `#4A6FA5` (청색) | 산업혁명·과학의 이미지 |
| 현대 (contemporary) | `#7B73FF` (보라) | 디지털·현대성의 이미지 |

- **학습자 설명**: "마커의 색상은 해당 사건이 속한 시대를 나타냅니다. 같은 색의 마커는 같은 시대의 사건입니다."

#### ✅ 지구본 시각화 설계 이유

- **점(Point) 방식 채택 이유**: 영역(폴리곤) 표현은 역사적 국경의 불확실성으로 인해 오해를 유발할 수 있음. 점은 "이 사건이 대략 이 지역에서 발생했다"는 최소한의 공간 정보를 제공.
- **교육적 장점**: 학습자가 여러 사건의 공간 분포를 한눈에 파악할 수 있으며, 시대별 색상으로 시간 차원도 동시에 인지 가능.

#### ✅ 한계 및 주의사항

- 점 마커는 사건의 정확한 발생 범위를 나타내지 않음 (예: 임진왜란은 한반도 전역이나 대표 좌표 1개로 표시)
- 동일 좌표에 복수 사건이 겹칠 수 있음 → 클러스터링으로 해결

### 4.2 레이블 (Label)

#### ✅ 표현 로직

| 항목 | 구현 기준 |
|------|----------|
| 텍스트 구성 | `{아이콘 이모지} {현재 언어 제목}` |
| 폰트 | `13px Pretendard Variable, Inter, sans-serif` |
| 가시성 제어 | `scaleByDistance(1e6, 1.0, 8e6, 0.4)` — 줌 아웃 시 축소 |
| 투명도 제어 | `translucencyByDistance(1e6, 1.0, 1.2e7, 0.0)` — 1200만m 이상에서 투명 |
| 오프셋 | `pixelOffset(0, -18)` — 마커 위쪽에 배치 |

- **설계 이유**: 줌 아웃 시 레이블이 겹쳐 가독성이 떨어지는 것을 방지하기 위해 거리 기반 스케일·투명도 조절 적용
- **학습자 설명**: "지구본을 확대하면 사건 이름이 크게, 축소하면 자동으로 작아지거나 사라집니다."

### 4.3 클러스터링 (Clustering)

#### ✅ 표현 로직

| 항목 | 구현 기준 |
|------|----------|
| 활성화 | `clustering.enabled = true` |
| 클러스터 범위 | `pixelRange = 45px` |
| 최소 그룹 크기 | `minimumClusterSize = 2` |
| 클러스터 표시 | 원형 점(26px, `#4A6FA5` 85% 불투명) + 개수 레이블 |
| 클러스터 클릭 | 클릭 지점 기준 현재 카메라 고도의 절반으로 줌인 (최소 100,000m) |

- **설계 이유**: 동일 지역의 다수 사건이 축소 시 겹치는 문제를 해결. 클릭 시 단계적 줌인으로 자연스러운 탐색 경험 제공.
- **학습자 설명**: "숫자가 표시된 원은 그 지역에 여러 사건이 모여 있다는 뜻입니다. 클릭하면 확대됩니다."

### 4.4 선택 강조 (Pulse Animation)

#### ✅ 표현 로직

| 항목 | 구현 기준 |
|------|----------|
| 트리거 | 타임라인 또는 마커에서 아이템 선택 시 |
| 카메라 이동 | `flyTo(lng, lat, 3,000,000m)` — 1.5초 애니메이션 |
| 펄스 효과 | 1.5초 주기로 10px→50px 확대, 빨간색(`#EF4444`) 불투명도 0.8→0 페이드아웃 반복 |
| 계산 방식 | `CallbackProperty`로 매 프레임 `Date.now() % 1500`을 통해 0~1 사이의 값 생성 |

- **설계 이유**: 정적 마커만으로는 선택된 항목의 위치를 직관적으로 인지하기 어려움. 펄스 효과로 시선을 자연스럽게 유도.
- **학습자 설명**: "사건이나 인물을 선택하면 지구본이 해당 위치로 이동하고 빨간색 파동이 위치를 알려줍니다."

### 4.8 필터링 시스템

#### ✅ 표현 로직

필터 변경 시 `filteredItems`가 재계산되고, 지구본의 마커가 **전체 교체**됨.

| 필터 | 대상 필드 | 방식 |
|------|----------|------|
| 시대 | `item.era` | 다중 선택 (OR) |
| 지역 | `item.location.region` / `item.birthPlace.region` | 다중 선택 (OR) |
| 카테고리 | `item.category` | 다중 선택 (OR) |
| 유형 | `item.type` | 단일 선택 (all/event/person) |
| 중요도 | `item.importance` | 최소값 이상 |
| 키워드 | `title`, `description`, `location.name` | 부분 일치 검색 |
| 미디어 유형 | `relatedMedia` → `media.type` | 다중 선택 (OR) |

- **학습자 설명**: "필터를 사용하면 관심 있는 시대·지역·분야의 사건만 지구본에 표시할 수 있습니다."

## 6. 책임성과 교육적 재현성

### 6.1 동일 조건 재현

- 동일 데이터(`events.json`, `people.json`)와 동일 코드(`GlobeView.jsx`)를 사용하면 **동일한 시각화 결과**를 재현할 수 있음.
- CesiumJS의 렌더링은 GPU에 따라 미세한 차이가 있을 수 있으나, 마커 위치와 색상은 동일하게 보장됨.

### 6.2 다른 사료 적용 시 달라질 수 있는 요소

- 좌표: 사료에 따라 사건 발생 지점의 해석이 달라질 수 있음
- 시대 구분: 서양사 기준(476년, 1453년)이 아닌 한국사 기준을 적용하면 분류가 달라질 수 있음
- 중요도: 편찬 관점에 따라 `importance` 값이 달라질 수 있음

### 6.3 학습 활용 책임 범위

> 본 시각화는 역사 학습을 **지원**하기 위한 도구이며, 학술 논문이나 법적 근거로 사용할 수 없습니다. 역사 자료와 설정된 기준에 따라 시각화되었으며, 역사 해석의 다양성을 존중합니다.

## 7. 학습자용 요약 (Executive Summary)

### 이 지구본 시각화가 전달하는 핵심 메시지

1. **역사는 공간 위에서 일어납니다** — 사건과 인물은 특정 장소에서 발생하며, 지구본 위에서 그 위치를 확인할 수 있습니다.
2. **시대는 색으로 구분됩니다** — 황금(고대), 갈색(중세), 녹색(근세), 청색(근대), 보라(현대)로 시간을 시각적으로 인지할 수 있습니다.
3. **사건은 서로 연결되어 있습니다** — 관련 사건·인물·미디어를 통해 역사의 맥락을 파악할 수 있습니다.

### 학습 시 반드시 기억해야 할 관점

- 마커의 위치는 사건의 **대표 지점**이며, 실제 영향 범위는 더 넓을 수 있습니다.
- 지명 레이어는 **현대 지명**이므로 역사적 지명과 다를 수 있습니다.
- 시대 구분은 **서양사 기준**으로 설정되어 있으며, 한국사의 시대 구분과 완전히 일치하지 않을 수 있습니다.
- 이 시각화는 역사의 **한 가지 해석**을 보여주며, 절대적 진리가 아닙니다.
